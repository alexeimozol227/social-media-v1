# 07-monetization.md — Тарифы, лимиты и подписочная модель

> **Документ:** `07-monetization.md`
> **Назначение:** Зафиксировать ценовую модель, состав тарифов, политику лимитов, поведение upgrade / downgrade / cancel. Этот документ — продолжение `06-roadmap.md`: он отвечает на вопрос «что юзер платит и что за это получает». Прогнозов выручки, CAC / LTV / MRR / ARR здесь нет — это другой жанр документации.

## ⭐ Зафиксированные решения

> Нумерация продолжает сквозную линию: `01` (D1–D7), `02` (D8–D11), `03` (D12–D27), `04` (D28–D67), `05` (D36–D43), `06` (D44–D49). В `07` — **D50–D55**.

| # | Решение | Значение |
|---|---|---|
| D50 | **Три тарифа, цены за бренд-слоты** | **Solo / Pro / Network** — отличаются числом брендов, постов и AI-генераций. Это не пакеты с одинаковым набором фич, а ступени по объёму работы |
| D51 | **Annual = −17 %** | Годовая подписка стоит 10 месячных платежей (≈ −17 %). Оплачивается одним списанием, не разбивается на 12 траншей |
| D52 | **Free trial 7 дней без карты** | Полный доступ к **Pro** на 7 дней, без подключения платёжного метода. Карта запрашивается только когда юзер решил продолжать |
| D53 | **Дефолт UI — Monthly** | На странице тарифов по умолчанию выбран Monthly; рядом виден toggle «Annual − save 17 %». Без dark patterns со скрытым переключателем |
| D54 | **Лимиты: soft + hard, без pay-per-use** | **Posts / AI text / AI media** — soft (банер от 80 %, блок после 100 % с 24 ч grace); **бренды / каналы / конкуренты** — hard cap всегда (анти-abuse). **Overage / pay-as-you-go не вводим ни в каком виде** — юзер либо в лимите, либо апгрейдится. Boost-паки — возможная post-MVP опция (v1.2+) при подтверждённом запросе пилотов, и тоже не как счётчик расхода, а как фикс-покупка |
| D55 | **Ожидаемое распределение тарифов** | **40 % Solo / 50 % Pro / 10 % Network** — это **design assumption**, по которой балансируется состав фич и лимитов между тарифами, не прогноз выручки |

## 1. Принципы ценообразования

| Принцип | Что значит |
|---|---|
| **Value-based, не cost-plus** | Цена ставится от пользы (сэкономленные часы × внутренняя ставка юзера), а не от наших затрат на токены. Один час ручной работы редактора в нише ≥ цена месячного Pro |
| **Простая структура** | 3 тарифа, без add-on'ов на MVP. Юзер выбирает за 30 секунд |
| **Annual = −17 %** | Годовая подписка — 10 месячных платежей. Оплачивается одним списанием |
| **Валюта цены — по стране юзера** | Россия — `₽` (RUB), Беларусь — `Br` (BYN), все остальные — `$` (USD). Страна определяется по гео-IP при первом заходе на лендинг + перепроверяется по BIN-карте в момент оплаты. Юзер всегда видит **одну** строку прайса в своей валюте — без переключателей RUB↔USD. Цена в каждой валюте — фиксированная (не пересчитывается по курсу ежедневно), пересмотр — 1 раз в квартал. Конкретный платёжный шлюз внутри `PaymentProvider`-абстракции (D21 в `03`, `04 §6` / `05 §7`) выбирается по карте юзера, юзера это не касается |
| **Free trial 7 дней без карты** | Снижает барьер: юзер успевает подключить канал, прогнать Onboarding Agent и увидеть первые черновики до того, как платит |
| **Прозрачные лимиты** | Что входит в тариф — видно на странице тарифов и в кабинете (F8b в `03`). AI-визуалы (картинки) — отдельной строкой, потому что это заметная статья COGS |
| **No early-adopter discounts на старте** | До публичного запуска — полная цена. Beta-программа (50 % off на год для первых 50) включается только в Фазе 6 (D47 в `06`): продаём ценность, а не дисконт |
| **Подписочная модель, не pay-per-use** | См. D54. Внутренняя cost-телеметрия (П12 в `04`, D37) видна только админу — F8 (Cost Observability в admin-панели). В клиентском UI расходометра НЕТ |

### 2.2 Что входит во все 3 тарифа без исключений

- **8 MVP-агентов** (D24 в `03`):
  - **Content** (Heavy, Claude Sonnet 4.6) — генерация постов
  - **Publisher** (Light, GPT-4o-mini) — публикация с idempotency + подбор времени
  - **Analyst** (Medium, Gemini 2.5 Pro) — еженедельный AI-отчёт
  - **Moderation** (Light, GPT-4o-mini) — pre-publish фильтр (rule + LLM-judge)
  - **Orchestrator** (Light, GPT-4o-mini) — координация цепочки
  - **Brand Memory** (Light, GPT-4o-mini) — фоновое обновление BM
  - **Onboarding** (Medium, Claude Haiku 4.5) — авто-экстракция BM ≤ 15 мин
  - **Notification** (Light, GPT-4o-mini) — TG-бот для оперативных уведомлений (J1–J4), email через `EmailProvider` для биллинга (J5) и еженедельного отчёта (J6)
- **Media — пайплайн внутри Content** (`04 §7.1`): когда Content-агент решает, что посту нужен AI-визуал, внутри пайплайна вызывается `ImageProvider`. **Отдельным 9-м агентом Media не считается** — пользователь видит «Content сгенерировал пост с картинкой»
- **Brand Memory** двухслойная: Core Profile + Network Overlays (D33 в `04`)
- **Cold-start wizard** для каналов с < 10 постов
- **Базовая аналитика** канала (просмотры, реакции, шеры, динамика подписчиков)
- **Audit Log + AI Explainability UI** (П5 в `04` — «как агент это сделал»)
- **Kill-switches** на стороне юзера: можно поставить на паузу Full-auto, AI-визуалы, отдельный канал (`04 §16.7`)
- **TG-бот approve flow** через MiniApp (S5 в `03`)
- **Tiptap-редактор + TG MarkdownV2** (D17 в `03`)
- **Календарь публикаций с drag-and-drop** (C6 в `03`)
- **Approve mode + Full-auto mode** (Full-auto всегда с safety-net Moderation Agent, П10 в `04`)
- **Контент-план на неделю** — Content Agent в batch-режиме на MVP; через отдельный `StrategistAgent` — post-MVP v1.1
- **Еженедельный AI-отчёт** — Analyst Agent, доставляется в TG-бот (J6) с дублированием в email через `EmailProvider`
- **Самописная авторизация:** JWT (15 мин access) + refresh-токен (30 дней, HttpOnly cookie с ротацией) + email/password + email verification + **MFA (TOTP)**, экспорт данных, удаление аккаунта (П7 в `04`)

### 2.3 Что НЕ входит ни в один тариф

- Прямой доступ к `LLMProvider` (юзер не делает raw-запросы к Anthropic / OpenAI / Gemini — это внутренняя инфраструктура)
- Исходный код агентов / промптов / skill manifest-файлов
- Кастомные интеграции (это уровень Agency / Enterprise post-MVP, см. §9)
- **Post-MVP агенты** (Engagement, Research, Strategist, Optimizer, Monitor, Repurpose) — добавляются поэтапно по `06 §10–11`, часть из них приходит в P1-пакете в месяц 7, остальные — в v1.1–v1.6

## 3. Free trial — 7 дней без карты (D52)

**Правила:**

- Без карты на старте — снимаем максимальный барьер входа (D14 в `03`)
- Полный доступ к **Pro**-тарифу на 7 дней (3 бренда, 100 постов, 100 AI-визуалов)
- На 6-й день — TG-бот напоминание (J5-аналог) + email через `EmailProvider`
- На 7-й день — workspace переходит в `read-only`: данные сохраняются, новые AI-генерации недоступны, запланированные публикации **останавливаются** (флаг `auto_publish_enabled=false` в Unleash)
- На 14-й день — ещё одно email-напоминание с возможностью реактивации
- Через 30 дней read-only без активации — workspace архивируется (не удаляется); данные хранятся 6 месяцев

**Почему без карты:** activation (signup → first post) важнее, чем conversion из trial. Карта на старте режет conversion. К концу trial у юзера уже подключён канал, заполнен Brand Memory и опубликованы посты — sunk-cost работает на удержание.

**Anti-abuse:** 1 email = 1 trial. Дополнительно — Telegram chat_id привязывается через `/start <user_token>` deep-link. Многократные регистрации с одного устройства / IP лимитируются через `slowapi` rate-limit на бэкенде + Unleash kill-switch (`04 §17.3` + D58). `Idempotency-Key` (П13 в `04`) защищает от дублирующих активаций.

## 4. Annual vs Monthly (D51, D53)

> Числа ниже — в USD для справочной полноты. Юзер из России видит те же `× 12` расчёты в `₽`, юзер из Беларуси — в `Br`. Логика «10 месячных платежей за год» одинаковая во всех трёх валютах.

| | Monthly ($/год) | Annual ($/год, −17 %) |
|---|---|---|
| Solo | $18 × 12 = $216 | **$15 × 12 = $180** (экономия $36) |
| Pro | $42 × 12 = $504 | **$35 × 12 = $420** (экономия $84) |
| Network | $95 × 12 = $1 140 | **$79 × 12 = $948** (экономия $192) |

**Annual оплачивается одним списанием.** Это снижает churn (юзер не «забывает» отменить помесячку) и улучшает кеш-флоу.

**Дефолт UI (D53):** toggle `Monthly | Annual` на странице тарифов, по умолчанию активен `Monthly`. Рядом виден badge «Save 17 % with annual» — выгода видна, но юзер сам решает, без скрытого переключателя.

## 5. Лимиты: что делаем при превышении (D54)

### 5.1 Soft limits — предупреждаем, но не блокируем сразу

| Лимит | 80 % | 100 % |
|---|---|---|
| Постов / мес | Жёлтый банер «80 % постов израсходованы. Перейти на Pro» | Блокирующий банер: «Лимит исчерпан». Действие выполняется в течение **24 ч grace**, потом блокируется до следующего месяца / upgrade'а |
| AI text-генераций | То же | То же |
| AI media-генераций | То же | То же |

- Все уведомления о лимитах — **только TG-бот** (J1–J4 в `04 §8.6.1`). Email — только для биллинга (J5) и еженедельного отчёта (J6)
- В клиентском UI отображается **прогресс-бар лимитов в постах и AI-генерациях** (F8b в `03`) — не cost-счётчик, не токены, не рубли. Только «осталось X постов / Y AI-визуалов в этом месяце»

### 5.2 Hard limits — никогда не превышаем

| Лимит | Поведение |
|---|---|
| **Брендов** | Физически нельзя добавить больше, чем по тарифу. Защита от abuse и нагрузки на Brand Memory cache |
| **TG-каналов на бренд** | Аналогично |
| **Конкуренты на мониторинг** | Аналогично; уже добавленные продолжают парситься через Global Channel Registry (D20) — нагрузка не растёт, потому что один канал = одна запись для всех тенантов |
| **Cost budget per brand** (D66 в `04 §16.7`) | Трёхступенчатый внутренний cap, синхронизированный с **CostGuardian** (D59 в `04 §16.6`) — см. §5.3 |

### 5.3 Cost budget per brand — 3 ступени

> Внутренний механизм CostGuardian (F8 в `03`); пользовательский эффект через F8b (блокируется именно автопубликация для конкретного бренда; остальное продолжает работать).

| % MTD-затрат бренда / `(подписочная цена × 0.6)` | Действие | Кто узнаёт |
|---|---|---|
| **60 %** | Алерт в админ-панель (Sentry `cost_budget_60pct`) | Только `admin` / `support` (юзер не видит) |
| **80 %** | TG-бот юзеру: «Использовано 80 % месячного лимита генераций. [Upgrade]». Email **не** дублируется | Юзер + админ |
| **100 %** | Kill-switch для бренда: `auto_publish_enabled=false`, manual approve остаётся. Реактивация — следующий billing-период или upgrade тарифа | Юзер + админ (severity=critical) |

### 5.4 Per-post Cost Guardian triggers (`04 §16.6`, D59)

Параллельно с месячным cap'ом работают per-post реакции CostGuardian — защита от runaway-LLM-вызовов (один внезапно дорогой пост не должен ждать конца месяца, чтобы быть пойманным):

| Уровень | Действие |
|---|---|
| **T1** | Sentry alert admin'у |
| **T2** | Auto-downgrade модели (Sonnet 4.6 → Haiku 4.5) на 24 ч + TG-бот юзеру «временно используем эконом-модель» |
| **T3** | Throttle auto-publish 60 мин + strict Moderation |
| **T4** | Kill-switch full-auto для бренда → approve-only |

Конкретные пороги T1–T4 — в `04 §16.6` (зависят от тарифа: на Solo пороги ниже, на Network — выше).

### 5.5 Overage / pay-per-use — нет ни в каком виде

- Юзер либо в лимите, либо апгрейдится / ждёт следующего месяца
- **Boost-паки** (`+50 AI-text` / `+20 AI-media`) — возможная post-MVP опция (v1.2+) при подтверждённом запросе пилотов; **не как pay-per-use счётчик**, а как фикс-покупка (один клик, фикс цена, фикс пакет, без долива «по факту расхода»)
- Дополнительные слоты бренда / канала **не продаются отдельно** на MVP — это меняет позиционирование тарифа. После v1.6 (Agency Mode) — пересмотрим в рамках Agency-tier

## 6. Upgrade / Downgrade / Cancel / Refund

### Upgrade

- Действует **немедленно**
- Биллинг — пропорциональный (prorated): юзер платит разницу за оставшиеся дни цикла
- Лимиты обновляются мгновенно
- `Idempotency-Key` защищает от двойного списания (П13 в `04`)

### Downgrade

- Действует **с начала следующего цикла** (без рефанда за текущий период)
- Если у юзера брендов больше, чем в новом тарифе, — UI заставляет выбрать, какие оставить активными; остальные переходят в `archived` (данные сохраняются)
- Если выбор не сделан — лишние бренды замораживаются автоматически через 7 дней с email-напоминаниями (J5)
- Brand Memory остаётся в «cold» и реактивируется при апгрейде

### Cancel

- Действует **до конца текущего периода** (юзер дочитывает оплаченное)
- После окончания — workspace в `read-only`, запланированные публикации останавливаются
- Через 30 дней `read-only` → `archived`
- Данные хранятся 6 месяцев, потом удаляются (Notification Agent шлёт email-предупреждение за 30 дней — J5)
- Экспорт данных доступен в любой момент (П7 в `04`)

### Refunds

- По умолчанию **рефанды не делаем**
- Исключения: технические сбои на нашей стороне (downtime > 24 ч, документированный в Sentry), баг, который не давал использовать продукт
- Refund policy — отдельный пункт в User Agreement; исполняется через ручной процесс из admin-панели (M3 в `03`, доступ только роли `admin` — `04 §17.2`)

## 7. Margin discipline (внутренний design-constraint)

> Этот раздел — **внутренний** design-constraint, по которому строится прайс. Это **не** investor finance forecast: никаких MRR / ARR / CAC / LTV прогнозов в `07` нет — они живут в отдельном внутреннем документе финплана.

### 7.1 Цель по валовой марже

| Тариф | Целевая Gross Margin |
|---|---|
| Solo | ≥ 75 % |
| Pro | ≥ 75 % |
| Network | ≥ 70 % |

Если фактическая Gross Margin тарифа падает ниже цели 2 недели подряд — это триггер на одно из:
- Ужесточение лимитов (D54)
- Маршрутизация AI-визуалов на более дешёвый `ImageProvider`-адаптер (премиум → бюджетный, `04 §6.1`)
- Включение prompt-caching на агентах (`04 §16.5`)
- Пересмотр цены тарифа в следующем billing-окне

### 7.2 Главные драйверы COGS

- **AI-визуалы (`ImageProvider`)** — основная переменная: пост с картинкой в 3–4× дороже текстового по COGS. На Solo по умолчанию бюджетный провайдер; на Pro / Network — премиум; в обоих случаях `ImageProvider` — абстракция (D40 в `05`), конкретный провайдер выбирается роутингом
- **LLM-вызовы (`LLMProvider`)** — основной агент по cost'у — Content Agent (Heavy, Claude Sonnet 4.6). Brand Memory / Orchestrator / Moderation / Notification / Publisher — Light (GPT-4o-mini), их доля незначительна
- **Email-канал (`EmailProvider`)** — копеечная статья при правильной маршрутизации (только J5 биллинг + J6 еженедельный отчёт); если эта статья растёт — значит, NotificationAgent ошибочно отнёс срочный alert в email вместо TG-бота (`04 §8.6.1`)
- **Хранилище медиа** — через `ObjectStorageProvider` абстракцию (`05 §7`); конкретный провайдер выбирается после Фазы 6
- **Платёжные комиссии** (`PaymentProvider`) — зависят от шлюза; целевое окно: 2.5–3.5 % при списании

> Конкретные числа COGS в долларах на одного юзера — во внутреннем документе `agent-cost-analysis.md`. Цель `07` — зафиксировать **политику** маржинальности, а не конкретные цифры COGS.

### 7.3 Worst-case (power-user, упирающийся в лимит)

На Pro-тарифе теоретически возможен power-user, который выбирает все 100 постов с AI-визуалом — его COGS близок к 40 % от цены подписки. Mitigation:

- Жёсткие лимиты (D54) не дают выйти за 100 постов / 100 визуалов в месяц
- Прогресс-бар лимитов в UI (F8b) — не cost-счётчик, а quota: юзер видит, сколько осталось, без рублей / токенов
- При 80 % media-бюджета — авто-switch с премиум-`ImageProvider` на бюджетный с уведомлением «временно используем эконом-провайдер для визуалов»
- Prompt-caching (`04 §16.5`) снижает input-стоимость на 30–50 % на повторных контекстах Content Agent
- Outlier-юзеру (много генерирует, мало платит) — мягкое предложение Network через email-серию

## 10. Anti-abuse

- 1 email = 1 trial. 1 Telegram `chat_id` = 1 trial. Дополнительно — `slowapi` rate-limit + Unleash kill-switch для блокировки IP / Telegram-id при ботнетной активности (`04 §17.3`, D58)
- **`Idempotency-Key`** (П13 в `04`) на всех endpoint'ах с write-операциями — защита от дублирующих платежей и активаций
- **Bot-account detection** (post-MVP, v1.1): если за 1 час с одного устройства > 3 регистраций — каптча на 4-й
- **Audit Log** (`audit_events` таблица, M7 в `03`): все sensitive-операции (создание / удаление workspace, оплата, рефанд, impersonate admin'ом) логируются с `severity` (`info` / `warning` / `critical`)
- Marketplace post-MVP: модерация публикуемых пресетов и skill packs через `Moderation Agent` с дополнительным `marketplace-publication-validator` skill — защита от prompt-injection в виде skill'а

## 11. Marketplace (post-MVP, год 2)

- Юзеры (Pro / Network / Agency) публикуют **Brand Memory presets** (например, «Tone of voice для финансовых ниш») и **skill packs**
- Revshare: **70 % автору, 30 % платформе**
- Flywheel: больше пресетов / skill-pack'ов → больше юзеров → больше пресетов
- Технически — таблицы `marketplace_listings` / `marketplace_purchases` / `marketplace_authors`; payout идёт через тот же `PaymentProvider`, что и подписки (исходящие транзакции, не входящие)
- Все listing'и проходят `Moderation Agent` через `marketplace-publication-validator` skill (см. §10)
